## 二、关键逻辑说明

### 2.1 数据层（Entity + DAO + Migration）

**KnowledgePoint** 新增两字段：

```
subCategory: String = ""          → ""(经典角色) / "novel-character"(小说角色)
novelName: String = ""            → 归属小说名（仅 novel-character 使用）
```

**Room Migration 99→100**（`DatabaseMigrations.kt`）：
```sql
ALTER TABLE knowledge_points ADD COLUMN subCategory TEXT NOT NULL DEFAULT ''
ALTER TABLE knowledge_points ADD COLUMN novelName TEXT NOT NULL DEFAULT ''
```

**新增 DAO 查询**：
- `getBySubAndNovel(subCategory, novelName)` → 查某小说下的角色
- `getDistinctNovelNames(subCategory)` → 查不重复的小说名列表（用于二级分组）
- `getByCategoryNoSub(category)` → 查无子分类的条目（经典角色）

### 2.2 三级钻取管理页

```
一级（五个分类卡片）                    二级（仅人物可展开）          三级
┌─────────────────┐                  ┌─────────────────┐          ┌─────────────────┐
│ 人物  (12)       │ ──点击展开──→    │ 📌 经典角色 (3)  │          │ 林黛              │
│ 地点  (5)        │                  │ 📖 大宋 (7)      │ ──展开→ │ 赵佶              │
│ 事件  (3)        │                  │ 📖 夫人请住口 (2)│          │ 裴少卿            │
│ 笔记  (8)        │                  └─────────────────┘          └─────────────────┘
│ 其他  (2)        │
└─────────────────┘
```

- **一级**：默认显示 5 个分类，每个带计数
- **二级（仅人物）**：点击「人物」展开 → 经典角色（无子分类）+ 小说名分组（默认收起）
- **三级**：点击小说名分组展开 → 单张角色卡
- **收起**：点击人物分类行右侧的「⇱ 收起」→ 回到一级

### 2.3 编辑弹窗

- **新建**：点击右上角「+」
- **编辑**：点击已有条目（长按也触发）
- **删除**：编辑弹窗底部有「删除」按钮（仅编辑已有条目时显示）
- **人物子分类**：选「人物」大类后出现 RadioButton，选「小说角色」后出现小说名输入框

### 2.4 提取角色管线

```
用户选择章节（多选勾选）
       ↓
读取章节内容（每章截前 4000 字）
       ↓
分组送 AI 粗提炼（每批 3 段，并行×N）
       ↓
跳过失败的批次 ← 修复点：原版会插入"(分析失败)"污染合并
       ↓
合并摘要 → AI 提取角色 JSON
       ↓
JSON 解析 → 回退 Map<String, Any?>
       ↓
写入 KnowledgePoint DB（subCategory="novel-character"）
       ↓
Toast 提示提取结果
```

- **提示词**：`buildChunkBatchPrompt`（粗提炼）+ `buildMergePrompt`（合并提取），参考 Whisnya 的 `buildNovelChunkPrompt`/`buildNovelMergePrompt`
- **边界处理**：
  - 章节为空 → throw "请选择至少一个章节"
  - 读取失败某章 → 跳过不中断
  - API 超时/失败 → 目标批次跳过（不污染合并）
  - JSON 解析失败 → 返回空列表
  - 无角色 → Toast "未提取到任何角色"
  - 每章截 4000 字防超长

### 2.5 角色卡 @引用

在 AI Chat 弹 @知识点 弹窗时，小说角色卡子标题显示：
```
📖 大宋 · 林黛是个温婉聪慧的女子...
```

引用写入 ReferenceItem 时 title 自动带上前缀：`大宋/林黛`

### 2.6 提取角色 UI

- **入口**：AI Chat 右上角菜单「提取角色」（`ic_author` 图标）
- **章节选择**：全量 CheckBox 列表 + 快捷按钮（全选/取消/前20章）
- **默认**：选中前 20 章
- **进度**：按钮下方动态文字显示当前阶段
- **完成**：弹 Toast 含角色名 + 弹 AlertDialog 显示详细结果

### 4.3 提取角色

| 测试项 | 步骤 | 预期 |
|---|---|---|
| 无书籍进入 | 从「我的」页 → AI Chat → 提取角色 | Toast "请在阅读书籍时使用该功能" |
| 选择批量章节 | 点击 1/5/10/30 章（CheckBox）| 选中计数动态更新 |
| 全选/取消 | 点击快捷按钮 | CheckBox 同步 |
| 提取中吐司进度 | 点击「确定」 | 显示「读取第 X 章」「AI 分析中 (X/Y)」 |
| AI 返回有效角色 | 章节内容包含角色 | 写入 DB + Toast 显示角色名 + 弹窗详情 |
| AI 返回空角色 | 章节无明确角色（如说明文） | Toast "未提取到任何角色" |
| API 失败 | 断网或 Key 无效 | Toast "提取失败：..."，不崩溃 |
| 中途取消 | 提取中快速返回 | 任务在后台执行（无法立即取消，属预期）|
| 章节读取失败 | 某章无缓存 → 跳过 | 提取继续，跳过该章 |

### 4.4 错误态 & 边界

| 测试项 | 步骤 | 预期 |
|---|---|---|
| 章节列表为空 | 书籍无章节 | Toast "未能读取章节列表" |
| 小说角色名为空后保存 | 见 4.2 行 | |
| 提取结果重复执行 | 同一批章节提取 2 次 | 写入 2 条不同 id 的条目（可后续去重） |
| DB 版本降级 | 回退到旧版本 APK | Room 自动抛异常 → 用户重装（标准行为） |
